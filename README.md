# 好策自动阅读脚本

## 项目介绍

这是一个基于 Auto X JS 的自动化阅读脚本，专为好策APP设计，理论上也可用于其他阅读类应用。脚本通过模拟人类阅读行为，实现自动翻页、智能章节切换等功能，对高校的强制阅读书本毕业说拜拜

## 功能特性

- **自动翻页阅读**：模拟真实手指滑动，支持自定义阅读速度
- **智能章节切换**：自动检测页面底部，智能切换到下一章
- **人性化阅读**：曲线滑动轨迹，随机化参数，避免被检测
- **实时监控**：悬浮窗显示阅读状态，支持动态调整速度
- **智能底部检测**：基于多种检测方法，确保准确识别章节结束

## 技术实现

- **曲线滑动**：使用正弦函数生成平滑曲线，模拟真实手指滑动轨迹
- **智能底部检测**：
  1. 图片相似度对比（滑动前后截图对比）
  2. 底部阴影检测
  3. 滚动条位置检测
- **随机化参数**：每次滑动的距离、速度、等待时间均随机变化，模拟人类阅读行为
- **多线程处理**：使用线程实现倒计时显示，避免阻塞主线程

## 安装与使用

### 依赖要求

- **Auto X JS**：请使用 `aiselp/AutoX` 版本，不要使用官方社区 v6 版本
  - 仓库地址：`https://github.com/aiselp/AutoX`
- **Android 设备**：支持 Android 7.0 及以上系统
- **截图权限**：需要授予应用截图权限

### 为什么使用 aiselp/AutoX 而不是官方社区 v6 版本？

官方社区 v6 版本（autox-community/AutoX）存在屏幕内容识别问题，具体表现为：

1. **API 变更**：官方 v6 版本对底层 API 进行了较大改动，导致部分屏幕操作和图像识别功能不稳定
2. **兼容性问题**：脚本中使用的 `images` 模块和截图相关功能在官方 v6 版本中可能无法正常工作
3. **识别率低**：官方 v6 版本的图像识别精度下降，影响底部检测的准确性

而 `aiselp/AutoX` 版本保持了与脚本兼容的 API 实现，确保所有功能正常运行。

### 安装步骤

1. **下载并安装 Auto X JS 应用**：
   - 访问 `https://github.com/aiselp/AutoX` 仓库
   - 找到 `release` 页面
   - 下载 `Autox-v7-arm64-v8a-release-vx.x.x.apk` 文件（实际测试使用的是 `Autox-v7-arm64-v8a-release-v7.2.1.apk`）
   - 安装到手机，由于签名问题可能会报毒，请忽略，具体原因请查看仓库作者的详细说明
   - 开启 Autox 的无障碍服务（无需 Root 权限）

2. **下载并导入脚本**：
   - 下载本脚本文件 `haoce_auto.js`
   - 打开 Auto X JS 应用，导入下载好的脚本
   - 安卓设备上下载的脚本文件路径一般在 `Download` 文件夹中

**注意**：本脚本仅支持 Android 设备，由于 aiselp/AutoX 不支持 iOS，且 iOS 权限更严格，未来也不会有适配计划。

### 使用方法

1. 打开好策APP，手动选择待阅读的书本
2. 进入阅读页面后，在 Auto X JS 应用中运行脚本
3. 选择阅读速度（慢速/正常/快速）
4. 点击"开始"按钮开始自动阅读
5. 拖动悬浮窗标题可移动位置
6. 阅读完成后，点击"停止"按钮结束脚本
7. 手动切换到下一本图书，重复上述步骤

## 阅读速度配置

| 模式 | 下滑行数 | 阅读时间 | 适合场景 |
|------|---------|---------|----------|
| 慢速 | 3-5行 | 20-30秒 | 深度阅读，学习内容 |
| 正常 | 5-8行 | 14-22秒 | 日常阅读，推荐使用 |
| 快速 | 8-12行 | 8-14秒 | 快速浏览，情节回顾 |

## 已知问题与限制

1. **屏幕底部检测规则**：底部阴影检测和滚动条位置检测存在问题，大部分情况下不会被触发，识别率较低。但目前仅依靠图片相似度对比（滑动前后截图对比），就已经足够达到99%场景下的识别率，使用没有问题。

2. **兼容性问题**：不同设备的屏幕分辨率和阅读应用界面可能会影响脚本的准确性，建议在使用前进行适当测试。

3. **权限要求**：需要授予截图权限和无障碍权限，部分设备可能会提示权限申请。
